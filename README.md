## Workflow for dk2nu_to_marley studies

### Running dar-enabled dk2nu files through GENIE

The first method consists of running dk2nu (with uDAR and piDAR enabled) generated g4bnb files generated by me through the GENIE generator following Marco del Tutto’s workflow idea for similar beam dump studies.

The flux files being used are (hardcoded into fhicl):

```bash
/pnfs/sbnd/persistent/users/lucykots/G4BNB_muonDAR/1000n_5000p_pidar_udar/production_muDAR_piDAR_BooNE_50m_I174000A/all_NuBeam_files_fhc/

#Example files within above directory:
NuBeam_production_muDAR_piDAR_BooNE_50m_I174000A_*.dk2nu.root
```

Commands to produce events from above dk2nu files running through GENIE:

```bash
sh /exp/$(id -ng)/data/users/vito/podman/start_SL7dev.sh #setup container SL7 on gpvm                                                                                    

cd /exp/sbnd/app/users/lucykots/dk2nu_to_marley/

#Setup larsoft
source /cvmfs/sbnd.opensciencegrid.org/products/sbnd/setup_sbnd.sh                                                                                                       
source localProducts_larsoft_v10_15_00_prof_e26/setup
mrbsetenv
mrbslp

#Run fhicl and make files!
cd outputs/highstats/
#for alln run lar -c ... -n -1
lar -c prodgenie_nu_spill_tpc_lowEen_sbnd_inclusive.fcl -n ${events} -o /pnfs/sbnd/persistent/users/lucykots/g4bnb_to_gen_DAR/dk2nu_to_genie/outputs/gen/alln/prodgenie_nu_spill_tpc_lowEen_sbnd_alln.root

#after running will be in /pnfs/sbnd/persistent/users/lucykots/g4bnb_to_gen_DAR/dk2nu_to_genie/outputs/gen/alln/
#Then run cafs

#Turn files into flatcafs
lar -c cafmakerjob_gen_sbnd.fcl -s /pnfs/sbnd/persistent/users/lucykots/g4bnb_to_gen_DAR/dk2nu_to_genie/outputs/cafana/prodgenie_nu_spill_tpc_lowEen_sbnd_${events}.root
```

### Running dar-enabled dk2nu files through MARLEY (Steven Gardiner’s method)

This method consists of running dk2nu (with uDAR and piDAR enabled) generated g4bnb files generated by me through MARLEY masking as genie. The repositories sbndcode, dk2nu and genie are all involved in this build developed by Steven Gardiner. I have simply copied his build and changed his paths to my paths in the setup file:

```bash
#(Only needed to be done onceto setup)

#Working directory
cd /exp/sbnd/app/users/lucykots/DK2NU_MARLEY_GENIE_hack/steven_build/
cp /exp/sbnd/app/users/gardiner/mudar-dev-new/ .
emacs -nw localProducts_larsoft_v10_06_00_e26_prof/setup #change paths here
```
